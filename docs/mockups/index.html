Visual direction — pick one

Same UI snapshot in three styles: the Compose step (Step 3) of the Apply wizard, two recommended experiences with bullets, drawer affordance, footer actions. All three are dark-mode, vanilla HTML+CSS, no JS interactions (visual comparison only).

AStep-zone palette

Closest to LCARS philosophy: color does wayfinding. Each wizard step has its own muted accent (Job=blue, Clarify=violet, Compose=teal, Template=amber, Generate=orange, Download=green). The active step's accent fills the left border, headers, and rail indicator. Semantic chips (recommended / outcome / pin / exclude) overlay it. Antonio is kept for headers + numerals as a typographic nod.

→ Open mockup A

BOne brand accent + functional palette

Linear / Notion-ish discipline. Single restrained accent (refined amber) for active state and brand. All wayfinding via numbered stepper + active state. Functional color reserved strictly for status (green=success/recommended, amber=brand/active, red=destructive, blue=info/link). System font everywhere, sentence case body, ALL CAPS only on chips ≤ 8 chars. Lowest visual noise.

CHybrid: brand accent + step-zone tint

Best of A and B. Neutral surfaces and signature amber for active/primary buttons (B's discipline), AND each step panel carries a thin 3px top-border tint in its zone color (A's wayfinding). The active rail step's indicator gently pulses in its zone tint (reduced-motion safe). System body type with Antonio reserved for the app title + rail numerals. Recognizable LCARS lineage without LCARS chrome.

→ Open mockup C

How to pick:
  • A if you want color to do most of the wayfinding work — the most distinctive identity, the most colorful surface, the closest to LCARS's intent.
  • B if you want the most disciplined contemporary look — minimal color, maximum content focus, easiest to defend on accessibility/contrast.
  • C if you want both: a recognizable brand accent for action + just enough zone color to anchor where you are, without LCARS's color dominance.

Open all three in tabs and switch between them. The same data is rendered in each so the comparison is apples-to-apples. Once you pick, Branch 2 implements the chosen direction across Apply / Library / Settings.
